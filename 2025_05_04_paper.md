# Summary
<!-- Add Figure: Logo -->

`pycohortflow` is a small utility Python package for generating publication-ready cohort flow diagrams from a declarative description of cohort steps. It ships with a faithful JavaScript port with no runtime dependency and an interactive diagram generator in the documentation. `pycohortflow` aims to produce attrition diagrams commonly used in clinical research to show the studies underlying population flow through screenin, eligibility assesment, exclusion, follow-up and final analysis. 

The package is designed for reproducible manuscript workflows and interdisciplinary collaboration. The content layer (diagram description) and the styling layer (TOML-configuration) are completely separated. The Python backend integrates programmatically into analysis workflows and enables the production of cohort flow diagrams without overhead. # Statement of need
Standard reporting guidelines now exist for almost every major study design in the biomedical sciences, and most of them prescribe a participant flow diagram as a mandatory or strongly recommended figure: CONSORT 2025 for randomised trials [@Hopewell2025; @Hopewell2025EE], PRISMA 2020 for systematic reviews and meta-analyses [@Page2021], STROBE for observational studies in epidemiology [@vonElm2007], and TRIPOD+AI for prediction model studies [@Collins2024].
Additional information on the role of these guidelines can be found by @Ferreira2021.

Flow diagrams provide the fastest way for readers, peer reviewers, and editors to assess the two main threats to the internal validity of clinical and observational research: selection bias and attrition bias. Despite this near-universal recommendation, compliance with the flow-diagram reporting remains poor in practice. Audits across pharmacology, infectious-disease, and early-phase oncology trials report that a substantial fraction of published RCTs either omit the flow diagram entirely or fail to report key items such as reasons for exclusion at allocation or for loss to follow-up [@Godwin2019; @Sebille2019].
Additionally, a frequently cited driver of these errors is the manual creation of the figure, typically in a slide-editor or vector-graphics program: numbers are copied from analysis output by hand, boxes are added and removed in response to revision rounds, and any change to the underlying cohort definition has to be back-propagated into the figure. This also separates the figure from the analysis code that generated the underlying counts and thereby potentially breaking reproducibility. Furhtermore, it increases the chance of arithmetic errors and cnveying errors between analysis, text, tables, and figures.

Generating the diagram programmatically from the same data structure that drives the statistical analysis closes this loop and removes a class of preventable errors. `pycohortflow` addresses this narrow but frequent workflow problem for Python users who need single-pipeline cohort attrition diagrams. Its intended audience includes clinical researchers, physicians. epidemiologists, statisticians, and manuscript authors who already use Python for data processing or figure generation. + Interdisciplinary cooperations

Given that Python is currently becoming the dominant language for biomedical data science and machine learning, this gap imposes a recurring burden on small clinical and translational projects, which either fall back to manual graphics editors or pull in an entire R toolchain to render a figure of fewer than ten boxes. It integrates natively with commonly used Python Ecosystem: Jupyer Notebook and Matplotlib.

# State of the field
The space of code-driven flow-diagram tools is dominated by R packages, each making different trade-offs. `consort` [@Wang2023] and `ggconsort` [@Gerke2021] target CONSORT RCT diagrams; `flowchart` [@Satorra2025] constructs diagrams directly from a `data.frame` using a tidyverse pipe; `PRISMA2020` [@Haddaway2022] specialises in PRISMA two-column diagrams and exposes an interactive Shiny app. 
In Python, `equiflow` [@Estiri2024] is the closest existing package, but its purpose is to surface compositional shifts across exclusion steps for equity audits, not to render small, publication-ready figures from already-computed counts. Beyond these domain-specific tools, researchers commonly fall back to general-purpose diagram engines (Graphviz, draw.io, `DiagrammeR` [@IannoneDiagrammeR] or  slide editors such as PowerPoint or Inkscape), all of which require non-trivial layout work and are difficult to reproduce across revisions.

`pycohortflow` is complementary to these projects. Its distinguishing focus is a Python-native, Matplotlib-based implementation for single-arm cohort pipelines with a small dependency footprint and publication-oriented style configuration. Compared with R tools such as `consort` and `flowchart`, it integrates naturally into Python analysis notebooks, scripts, and multi-panel Matplotlib figures. Furhtermore, it provides a JavaScript interactive generator as no-code option. The same TOML configuration produces equivalent figures in both runtimes.
Currently, the package is deliberately scoped to small, single-column vertical diagrams; it does not attempt to compete with the multi-arm CONSORT support of `consort` or the two-column source-tracking of `PRISMA2020`, which keeps the API and the visual default sane for the most common case in small clinical and translational studies.

# Research Impact Statement
The immediate impact of `pycohortflow` is infrastructural: it converts a common reporting figure into a reproducible software artifact. This is directly relevant to research fields in which participant flow and attrition are already emphasized by reporting guidelines [@Schulz2010Consort; @Vandenbroucke2007Strobe; @Page2021Prisma]. By making the figure data-driven, the package helps align manuscript figures with analysis code and reduces opportunities for manual transcription or arithmetic errors.
Although the project is early stage, it is community-ready. It is open source under EUPL-1.2, installable from PyPI, fully documentaed and accompanied by a browser igenerator as no-code option. The repository has been used to produce the cohort flow diagrams in three manuscripts currently under review or in preparation. The list will be updated, as soon the manuscripts are made available.
